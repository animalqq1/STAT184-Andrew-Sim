# Project Plan Document

## 🎯 Goals
- Successfully complete the STAT184 assignment
- Learn and apply GitHub version control features
- Maintain a clean and organized repository

---

## 🧩 Needs
- GitHub repository setup
- Multiple branches (main and dev)
- Meaningful commits with proper messages
- Pull requests for merging changes
- Issue tracking system
- Clear documentation (README and plan file)

---

## 🚀 Steps

### Project Plan
1. Set up the GitHub repository
2. Create and update the README file
3. Practice version control concepts
4. Track progress using issues
5. Finalize documentation

### Repository Management Plan
1. Create a `dev` branch for development
2. Make multiple commits with clear messages
3. Use pull requests to merge into `main`
4. Resolve any conflicts during merging
5. Keep the main branch updated
6. Maintain organized and readable files

This plan ensures both the project work and repository management are clearly structured and completed effectively.
